## 1. 产品概述

### 1.1 产品定位

本项目是一款**高仿小米 MIUI 计算器**的微信小程序，面向微信生态用户，提供一体化的计算工具服务。产品不仅涵盖基础运算与科学计算，还集成了汇率换算、单位换算、个税计算、房贷计算等生活常用工具，打造「一个应用解决所有计算需求」的体验。

### 1.2 核心价值

| 维度 | 描述 |
|------|------|
| **便捷性** | 微信内即开即用，无需下载独立 App |
| **全面性** | 一个小程序覆盖日常计算、科学计算、生活换算全场景 |
| **一致性** | 完整还原小米计算器的交互逻辑与视觉风格，降低用户学习成本 |
| **轻量性** | 小程序包体 < 2MB，启动 < 2 秒 |

### 1.3 目标指标

| 指标 | 目标值 |
|------|--------|
| 首屏渲染时间 | < 1.5s |
| 基础运算响应 | < 100ms |
| 汇率数据更新 | 每日自动更新 |
| 小程序包大小 | < 2MB |
| 用户次日留存 | > 40% |

## 2. 用户分析

### 2.1 目标用户群

| 用户类型 | 场景 | 使用频率 |

| 普通消费者 | 日常购物、AA 制分账、折扣计算 | 每日多次 |

#### 5.1.2 按键定义

| 按键 | 功能 | 类型 | 颜色 |
|------|------|------|------|
| C / AC | 清空（短按清当前输入，长按全清） | 功能键 | 灰色 `#A5A5A5` |
| ← | 退格删除一个字符 | 功能键 | 灰色 `#A5A5A5` |
| % | 百分比转换 | 功能键 | 灰色 `#A5A5A5` |
| ÷ | 除法运算符 | 运算符 | 白色 `#FFFFFF` |
| × | 乘法运算符 | 运算符 | 白色 `#FFFFFF` |
| − | 减法运算符 | 运算符 | 白色 `#FFFFFF` |
| + | 加法运算符 | 运算符 | 白色 `#FFFFFF` |
| = | 计算结果 | 等号键 | 橙色 `#FF6900` |
| +/- | 正负号切换 | 功能键 | 白色 `#FFFFFF` |
| 0-9 | 数字输入 | 数字键 | 白色 `#FFFFFF` |
| . | 小数点 | 数字键 | 白色 `#FFFFFF` |

#### 5.1.3 交互逻辑

```javascript
// 计算逻辑状态机
states: {
  INITIAL: 'initial',           // 初始状态，显示 0
  INPUTTING_FIRST: 'inputting_first',   // 输入第一个操作数
  INPUTTING_SECOND: 'inputting_second', // 输入第二个操作数
  RESULT_SHOWN: 'result_shown',         // 显示计算结果
  ERROR: 'error'               // 错误状态
}
```

**核心规则**：
1. 连续按运算符时，以后者覆盖前者
2. 按 `=` 后如果继续按运算符，使用上一次结果继续运算
3. 除数为 0 时显示「错误」
4. 整数部分每 3 位自动添加千分位逗号
5. 结果最多显示 10 位有效数字，超出则使用科学计数法
6. 按 `C` 一次清除当前输入（显示为 AC → C 状态切换）
7. 百分比运算：`A + B%` 等价于 `A × (1 + B/100)`

### 5.2 科学计算器

#### 5.2.1 进入方式

- **方式一**：在基础计算器页面点击顶部「切换模式」按钮
- **方式二**：旋转手机横屏自动切换（监听 `deviceorientation`）
- **方式三**：从底部 Tab 切换至科学模式

#### 5.2.3 科学函数清单

| 类别 | 函数 | 说明 |
|------|------|------|
| 三角函数 | sin, cos, tan | 基于角度/弧度模式 |
| 反三角 | sin⁻¹, cos⁻¹, tan⁻¹ | 结果受 DEG/RAD 影响 |
| 对数 | log (log₁₀), ln (logₑ) | |
| 指数 | eˣ, 10ˣ, x², x³, xʸ | |
| 根号 | √, ³√ | |
| 阶乘 | n! | n 为非负整数 |
| 常数 | π (3.14159...), e (2.71828...) | |
| 模式 | DEG / RAD | 角度/弧度切换，影响三角函数 |

#### 5.2.4 DEG/RAD 切换

- 顶栏左侧显示当前角度模式
- 点击切换 DEG ↔ RAD
- 切换后，三角函数运算自动按新模式计算
- DEG 模式下：sin(30) = 0.5
- RAD 模式下：sin(π/6) = 0.5

#### 5.5.2 个税计算规则

**起征点**: 5,000 元/月（60,000 元/年）

**累进税率表（综合所得）**:

| 级数 | 累计预扣应纳税所得额 | 税率 | 速算扣除数 |
|------|---------------------|------|-----------|
| 1 | ≤ 36,000 | 3% | 0 |
| 2 | 36,000 ~ 144,000 | 10% | 2,520 |
| 3 | 144,000 ~ 300,000 | 20% | 16,920 |
| 4 | 300,000 ~ 420,000 | 25% | 31,920 |
| 5 | 420,000 ~ 660,000 | 30% | 52,920 |
| 6 | 660,000 ~ 960,000 | 35% | 85,920 |
| 7 | > 960,000 | 45% | 181,920 |

**计算公式**:
```
应纳税所得额 = 税前收入 - 五险一金 - 专项附加扣除 - 起征点(5000)
应纳税额 = 应纳税所得额 × 对应税率 - 速算扣除数
税后收入 = 税前收入 - 五险一金 - 应纳税额
```

#### 5.6.2 计算逻辑

**等额本息**:
```
月供 = [贷款本金 × 月利率 × (1+月利率)^还款月数] / [(1+月利率)^还款月数 - 1]
总利息 = 月供 × 还款月数 - 贷款本金
```

**等额本金**:
```
每月本金 = 贷款本金 / 还款月数
第n月月供 = 每月本金 + (贷款本金 - 已还本金累计) × 月利率
总利息 = (还款月数 + 1) × 贷款本金 × 月利率 / 2
```

## 11. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| 汇率 API 不稳定 | 汇率功能不可用 | 多备用 API 源 + 本地缓存 + 手动输入汇率 |
| 小程序审核不通过 | 延期上线 | 避免使用红包/诱导分享等敏感功能 |
| 科学计算精度问题 | 计算不准确 | 使用成熟的计算库（mathjs / decimal.js）或自行实现高精度算法 |
| 包体积过大 | 加载慢、审核困难 | 图片使用 WebP，代码分包加载，Tab 页懒加载 |
| 个税政策变化 | 计算结果错误 | 税率表抽离为配置文件，方便热更新 |
| iOS/Android 兼容性 | 部分机型异常 | 充分真机测试，覆盖 Top 50 机型 |
